Обзор всех вех — `roadmap.md`. Зависит от **вехи 1** (Accessibility-детект, разрешения,
foreground-сервис, DataStore).

## Цель вехи

Реализовать сердце родительского контроля на детском устройстве:
1. Точный учёт **реального экранного времени**.
2. Дневной лимит (по дням недели).
3. Белый список «всегда доступных» приложений.
4. Оверлей-блокировка по истечении лимита + предупреждение заранее.

Правила пока хранятся и настраиваются **локально** (синхронизация между устройствами — веха 4).

## Декомпозиция на шаги

### Шаг 2.1 — Доменная модель и Room
- Подключить **Room**. Сущности: политика ребёнка (дневной лимит по дням недели), белый список
  (пакеты «всегда доступны»), накопленное экранное время за день (день → минуты).
- Доменные модели + репозитории (Clean Architecture).
- **Критерий:** данные читаются/пишутся, схема мигрируется.

### Шаг 2.2 — Движок учёта реального экранного времени
- В foreground-сервисе (из вехи 1) — тикер, инкремент времени **только когда**: экран включён
  (`PowerManager.isInteractive`) И разблокирован (`KeyguardManager` / `ACTION_USER_PRESENT`) И
  есть активное приложение (из AccessibilityService).
- Слушать `ACTION_SCREEN_ON/OFF`, `ACTION_USER_PRESENT`. Музыка при погашенном экране — НЕ считать.
- Накопленное время писать в Room (с устойчивостью к перезапуску сервиса).
- **Критерий:** экранное время растёт только при активном использовании; фоновая музыка и
  погашенный экран время не расходуют (проверка сценарием).

### Шаг 2.3 — Проверка дневного лимита
- Сравнение накопленного времени за день с лимитом (учёт дня недели). Сброс в начале дня.
- Состояние «время вышло» / «осталось N минут».
- Белый список исключается из блокировки всегда.
- **Критерий:** при достижении лимита система переходит в состояние «время вышло».

### Шаг 2.4 — Оверлей-блокировка
- Когда активное приложение НЕ в белом списке И время вышло → блокирующий оверлей
  (`SYSTEM_ALERT_WINDOW`) поверх экрана + возврат на home. Белый список остаётся доступен.
- Оверлей информативный: «Время на сегодня закончилось».
- **Критерий:** после лимита все приложения кроме белого списка перекрываются оверлеем; телефон,
  SMS, контакты (белый список) работают.

### Шаг 2.5 — Родительский UI (локально)
- Экран настройки дневного лимита (по дням недели).
- Управление белым списком: список установленных приложений (`PackageManager`) с отметками.
- **Критерий:** родитель задаёт лимит и белый список, они применяются на устройстве.

### Шаг 2.6 — Предупреждение перед истечением
- За N минут до лимита — оверлей/уведомление «осталось N минут».
- **Критерий:** предупреждение появляется заранее, ребёнок не блокируется внезапно.

## Definition of Done

- Экранное время считается корректно (реальное использование, без фоновых процессов).
- Дневной лимит по дням недели работает; по истечении — блокировка всего, кроме белого списка.
- Родитель локально настраивает лимит и белый список.
- Предупреждение перед истечением работает.
