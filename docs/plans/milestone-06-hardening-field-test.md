## PIN — архитектура (через сервер, но БЕЗ правок сервера)

PIN родительский, задаётся в родительском приложении. Ключевая идея: **PIN едет в policy-документе** —
том же непрозрачном JSON, что уже синкается (как `bonuses`/`blockedApps`), поэтому **сервер трогать не
нужно** (он policy-agnostic).

- Родитель задаёт 4–6-значный PIN → клиент генерирует случайную соль и хеширует (`salt + PIN` →
  хеш, PBKDF2/SHA-256), кладёт `{pinHash, pinSalt}` в policy. **Сырой PIN нигде не хранится и на сервер
  не уходит — только хеш.**
- Синк доставляет хеш на детское устройство (Room), проверка ввода — **офлайн** на детском.
- PIN общий для семьи (оба родителя видят только хеш; кто задал последним — тот и действует, LWW).
  Забыл — переустановил в родительском приложении.

## Часть А — шаги

### Шаг 6.1 — PIN ✅
- Родительский экран установки/смены PIN (4–6 цифр, подтверждение). Хеширование с солью на клиенте,
  запись `{pinHash, pinSalt}` в policy-документ.
- Детское: чтение хеша из политики (Room), экран ввода PIN + офлайн-проверка (`hash(salt+ввод) == pinHash`).
- **Критерий:** PIN ставится у родителя, доезжает до детского, проверяется офлайн; сырой PIN не хранится.

### Шаг 6.2 — Защита настроек PIN-ом ✅
- `KidGuardAccessibilityService` (уже детектит активное приложение) ловит открытие критичных системных
  экранов — специальные возможности, VPN, «Администраторы устройства», «О приложении» KidGuard — и
  накрывает PIN-оверлеем: верный PIN → пропускает на короткое окно, иначе уводит назад.
- Список классов/пакетов критичных экранов собираем по факту (на HiOS уточним в части В).
- Вход в родительские настройки внутри детского приложения — тоже под PIN.
- **Критерий:** без PIN нельзя отключить accessibility/VPN/Device Admin.

### Шаг 6.3 — Защита от удаления (Device Admin) ✅
- Деактивация Device Admin (активен с вехи 1) и удаление KidGuard перехватываются тем же механизмом
  (6.2) → под PIN.
- **Критерий:** удалить приложение / отозвать Device Admin без PIN нельзя.

### Шаг 6.5 — Safe mode и известные пределы (документ) ✅
- Честно зафиксировать, что без Device Owner нельзя закрыть (safe mode — но там отключены и все сторонние
  приложения, стимул низкий; factory reset — Володя гарантирует не делать) и почему для нашего сценария
  это приемлемо. Прочие пределы.
- **Критерий:** ограничения честно задокументированы, критичных дыр для нашего сценария нет.
