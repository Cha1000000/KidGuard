# Веха 3Б — Бонусное время

> Составлено и согласовано с Владимиром 2026-07-08. Мини-веха, вынесена из вехи 3.
> Зависит от вех 2 (лимит/блокировка) и 3 (пер-app лимиты).

## Цель вехи

Родитель может **добавить ребёнку время сверх лимита на сегодня** — разовым бонусом, который
истекает в конце дня. Два вида:
- **Бонус на телефон** — прибавляется к общему дневному лимиту.
- **Бонус на приложение** — прибавляется к личному лимиту конкретного приложения.

Настройка локальная (Room); удалённая выдача с телефона родителя — веха 4.

## Согласованные решения

- **Оба типа** бонуса (телефон и приложение).
- **Суммируется:** повторная выдача в тот же день добавляется к текущему бонусу (15 + 15 = 30).
- **Срок:** только на сегодня, истекает в конце дня (граница дня — та же анти-отмотка через
  `CurrentDateProvider`, что и в вехах 2–3).
- **Отмена:** родитель может досрочно обнулить бонус (телефона или приложения).
- **Вход в UI:** бонус на телефон — блок/кнопка «Дополнительное время» на экране «Дневной лимит»;
  бонус на приложение — в bottom-sheet приложения на экране «Лимиты приложений» (уточняется макетом).
- **Название в UI:** блок называется **«Дополнительное время»** (не «Бонус на сегодня»). В коде и
  плане термин «бонус» остаётся как внутреннее имя сущности.
- **Ребёнку** отдельного UI не делаем (как в вехах 2–3) — бонус ощущается снятием блокировки.

## Ключевая идея (почему просто и без риска)

Бонус — это **дополнительные минуты к лимиту на сегодня**. Если прибавлять их прямо в расчёт
`LimitState`, то исчерпанный лимит (`Expired`) снова становится `Remaining`, и **матрицу
блокировки `shouldBlock` менять не нужно** — бонус вливается в use case'ы:

- `ObserveLimitStateUseCase`: `minutesLeft = (лимит + бонус_телефона) − использовано`.
- `ObserveAppLimitStateUseCase`: `minutesLeft = (лимит_приложения + бонус_приложения) − израсходовано`.

Бонус применяется только когда у цели есть лимит (без лимита блокировки нет — бонус ни к чему).

## Что уже есть (на что опираемся)

- `ObserveLimitStateUseCase` / `ObserveAppLimitStateUseCase` — расчёт остатка (веха 2/3).
- `CurrentDateProvider` — сегодняшняя дата с анти-отмоткой.
- Room `KidGuardDatabase` **version = 3** (после вехи 3); механизм явных миграций (`Migrations.kt`).
- UI-паттерны: экран «Дневной лимит» (пикер-слайдер в bottom-sheet), «Лимиты приложений»
  (bottom-sheet приложения) — бонусный пикер переиспользует их.

## Декомпозиция на шаги

### Шаг 3Б.1 — Модель бонусов (данные)
- Room `version 3 → 4` + **явная миграция**: таблица `bonus_grants`
  (`date` TEXT, `packageName` TEXT, `minutes` INT; PK = `date` + `packageName`).
  Маркер «весь телефон» — пустая строка `""` в `packageName` (в домене — `null`).
- Суммирование через upsert: `ON CONFLICT(date, packageName) DO UPDATE SET minutes = minutes + :minutes`.
- Новый доменный `BonusRepository` (в `:core`, реализация в `:data`):
  - `phoneBonusMinutes(date): Flow<Int>` — бонус телефона за день.
  - `appBonusMinutes(date): Flow<Map<String, Int>>` — бонусы приложений за день (пакет → минут).
  - `addBonus(date, packageName: String?, minutes: Int)` — добавить (null = телефон).
  - `clearBonus(date, packageName: String?)` — обнулить бонус цели.
- **Критерий:** бонусы сохраняются, суммируются, читаются; отмена обнуляет; миграция 3→4 без потерь.

### Шаг 3Б.2 — Применение бонуса в расчёте лимитов
- `ObserveLimitStateUseCase`: добавить в `combine` поток `phoneBonusMinutes(today)`; прибавлять
  к лимиту перед сравнением с использованным.
- `ObserveAppLimitStateUseCase`: добавить `appBonusMinutes(today)`; прибавлять к лимиту приложения.
- `shouldBlock` и `BlockingController` — **без изменений**.
- **Критерий:** при исчерпанном лимите выдача бонуса возвращает `Remaining` и снимает блокировку;
  в конце дня (смена даты) бонус перестаёт действовать.

### Шаг 3Б.3 — Родительский UI (⚠️ вне зоны кодинг-агента)
- Экран «Дневной лимит»: блок «Дополнительное время» — показывает текущий активный бонус (`+N мин`),
  кнопки «Выдать» (пикер минут) и «Отменить» (если бонус есть).
- Экран «Лимиты приложений», bottom-sheet приложения: секция бонуса для приложения (выдать/отменить),
  показ активного `+N мин`.
- **UI по правилу проекта: сперва макеты → согласование с Владимиром → реализация.** Кодинг-агент
  останавливается после 3Б.2 и возвращает управление на Opus.
- **Критерий:** родитель выдаёт/суммирует/отменяет бонус телефона и приложения; активный бонус виден.

## Definition of Done

- Бонус телефона и приложения выдаётся, суммируется в течение дня, отменяется досрочно.
- Выданный бонус реально снимает блокировку (телефона/приложения) до конца дня и истекает со сменой дня.
- Матрица приоритетов вехи 3 не сломана; суммарный и пер-app учёт не затронуты.

## Проверка

- **Юнит-тесты** обоих use case'ов с бонусом: `Expired` → `Remaining` при бонусе; суммирование;
  без лимита бонус не влияет (`NoLimit`).
- **Эмулятор** (техника вех 2–3): вставить лимит + расход до `Expired`, выдать бонус через UI/БД →
  блокировка снимается; сменить дату (анти-отмотка) → бонус не действует.

## Открытые вопросы / на потом

- Автоочистка старых `bonus_grants` (за прошлые дни) — не критично, можно фильтровать по `today`;
  чистку добавим при необходимости.
- Индикатор активного бонуса ребёнку — отдельная веха (детский UI).
