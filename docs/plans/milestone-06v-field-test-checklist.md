# Веха 6В — чек-лист обкатки на реальном Tecno Spark 30 Pro (HiOS)

> Рабочий документ для полевого теста. Составлен 2026-07-16, до получения устройства.
> Родительский план — `milestone-06-hardening-field-test.md` (шаги 6.4 и 6.6).
> Части 6А (защита от обхода) и 6Б (glassmorphism) влиты в `main` — обкатываем собранную систему.

## Решения перед стартом (Володя, 2026-07-16)

- **Релизный keystore НЕ заводим.** Приложение ставится вручную в своей семье, не в Play. Release
  подписан debug-ключом, и это не делает сборку debuggable (`debuggable` — отдельный флаг типа
  сборки). SHA-1 debug-ключа уже зарегистрирован в Google OAuth — вход работает. Заведём ключ, если
  когда-нибудь понадобится Play; тогда не забыть добавить новый SHA-1 в Google Cloud Console, иначе
  Google-вход сломается.
- **Обкатываем на debug-сборке.** На HiOS ожидаются вендор-специфичные сюрпризы, а `Timber` посажен
  только под `BuildConfig.DEBUG` (`KidGuardApp.kt`) — на release диагностика была бы вслепую.
  Перейдём на release, когда сценарии стабильно пойдут.
- **Устройства:** детское — Tecno Spark 30 Pro (HiOS), родительское — второй реальный телефон
  (или эмулятор, если второго нет). Сервер — боевой AdminVPS, debug-сборка уже смотрит на него.

### 0.1 Пакеты настроек и инсталлера

```bash
# Пакет системных настроек
adb shell cmd package resolve-activity --brief -a android.settings.SETTINGS | tail -1
# Пакет, который показывает диалог удаления приложения
adb shell cmd package resolve-activity --brief -a android.intent.action.DELETE -d package:ru.homelab.kidguard | tail -1
```

**Что ожидаем:** на эмуляторе — `com.android.settings/...` и
`com.google.android.packageinstaller/...`.

**Зачем:** если на HiOS инсталлер свой (`com.transsion.*` или иной) — **защита от удаления не
сработает**, пока детект завязан на список констант. Задача «убрать хардкод пакетов» это чинит
(определять через `PackageManager`), но факт всё равно надо зафиксировать: он проверяет, что фикс
угадал правильно.

### 0.2 Заголовки критичных экранов

Детект в `KidGuardAccessibilityService` идёт **по заголовку окна** (`AccessibilityWindowInfo.title`),
а не по классу активности — это сделано кросс-вендорно. Но ключевые слова надо сверить с фактом.

> ### ✅ Выполнено 2026-07-18 на ДЕТСКОМ Tecno Spark 30 Pro (KL6, Android 14, HiOS 14.5)
> **Важно:** HiOS **режет Timber-логи** сторонних приложений в logcat (видны только системные
> записи процесса) — команда с `KidGuardA11y` на этом устройстве бесполезна. Заголовки снимались
> через `dumpsys accessibility` (строки `A11yWindow[...title=...]`) — работает.
>
> | Экран | Заголовок на KL6 | Детект |
> |---|---|---|
> | Спец. возможности | **«Доступность»** | ❌ была дыра → ключ добавлен (`bad6999`) |
> | VPN | «VPN» | ✅ |
> | Админы (список) | «Приложения для администрирования устройств» | ✅ («администрир») |
> | Карточка админа KidGuard | «Активировать приложение администратора устройства?» | ✅ |
> | Дата и время | «Дата и время» | ✅ |
> | О приложении KidGuard | **«KidGuard»** (= label, не «о приложении»!) | ❌ была дыра → детект расширен на label (`bad6999`) |
> | Хранилище приложения | **«Память»** | ❌ была дыра → ключ добавлен (`bad6999`) |
> | Диалог удаления | «KidGuard» + пакет инсталлера | ✅ |
>
> Перехват «Доступности» и «О приложении» после фиксов **проверен живьём** на устройстве
> (PIN-оверлей появляется, верный PIN 2468 пропускает). Ещё факт: `am start` интентом настроек
> HiOS часто поднимает старую задачу настроек с последним открытым экраном — для чистого теста
> нужен `force-stop com.android.settings` + флаг `0x10008000` (NEW_TASK|CLEAR_TASK).

Открыть на устройстве по очереди и записать заголовок (на HiOS — через
`dumpsys accessibility | grep title=`, logcat бесполезен, см. выше):

| Экран | Ожидаемое ключевое слово | Факт на HiOS |
|---|---|---|
| Настройки → Спец. возможности | `специальные возможности` / `доступность` / `accessibility` | «Доступность» ✅ |
| Настройки → VPN | `vpn` | «VPN» ✅ |
| Настройки → Администраторы устройства (список) | `администрир` | ✅ |
| То же → карточка KidGuard | `администратор` | ✅ |
| Диалог деактивации Device Admin | `kidguard` + `отключ` | не проверялся (Device Admin ещё не выдан) |
| Долгий тап по иконке → Удалить | label приложения | «KidGuard» (инсталлер) ✅ |

**Если заголовок не совпал** — дописать ключевое слово в соответствующий `*_KEYWORDS` в
`KidGuardAccessibilityService`, НЕ переходя на детект по классу активности (правило: универсально,
не под одну модель).

### 0.3 Вендорный автозапуск (главный риск)

У Transsion есть **отдельный** менеджер автозапуска/энергосбережения помимо стандартного
`REQUEST_IGNORE_BATTERY_OPTIMIZATIONS`. Он умеет убивать foreground-сервисы, стандартного API нет.

> ### ✅ Выполнено 2026-07-18 на детском KL6
> `com.transsion.phonemaster/com.cyin.himgr.autostart.AutoStartActivity` (компонент, подтверждённый
> ранее на CM7) **существует и открывается интентом** → экран «Управление автозапуском».
> **KidGuard в списке уже «Разрешено»** (тумблер включён) — руками ничего делать не пришлось.
> Карточка мастера разрешений на этом устройстве откроет нужный экран сразу.

- Найти в настройках HiOS: «Автозапуск» / «Диспетчер автозапуска» / «Управление питанием» /
  «Защищённые приложения» — назвать может по-разному.
- Включить для KidGuard **вручную**, записать точный путь в настройках → он пойдёт в текст
  пункта мастера разрешений.
- Проверить, открывается ли этот экран интентом (мастер попытается, с фолбэком на настройки
  приложения).

### 0.4 Сопутствующие находки этапа 0 (2026-07-18)

- **На устройстве активен Google Family Link** (`com.google.android.gms.supervision`) — как
  **Profile Owner (user 0)** с полным набором policies. Уже форсит «Автоматическую установку
  времени» («Включено администратором» на экране «Дата и время» — перевод часов закрыт и без нас).
  Конкурирующий контроль; вопрос о его судьбе — Володе.
- **Device Admin у KidGuard не выдан** — мастер разрешений при первичной настройке завершён не до
  конца. Watchdog это честно показал у родителя: «⚠ Не работает: Администратор устройства» на
  карточке Олега — **первое полевое срабатывание watchdog на реальном устройстве**.
- У Олега установлен и включён в accessibility **Auto Clicker** — реальный подросток с реальными
  инструментами обхода; профильный контекст для вехи 6.
- Пароль экрана блокировки детского телефона — 2011 (сообщил Володя, для тестов после ребута).

## Этап 2 — Шаг 6.6: сквозные сценарии

**Критерий шага:** все сценарии проходят на реальных устройствах.

### 2.1 Учёт времени и дневной лимит
- [ ] Время тикает только при включённом+разблокированном экране (погасить экран → не растёт).
- [ ] Музыка в фоне при погашенном экране время НЕ ест.
- [ ] Родитель ставит лимит → доезжает на детский (WS-push, ~2-3 с).
- [ ] Лимит исчерпан → оверлей блокировки, закрывается только свайпом.
- [ ] Предупреждение «осталось N минут» приходит до истечения.
- [ ] Анти-отмотка: перевести время устройства назад → счётчик не сбросился.

### 2.2 Пер-app лимит и матрица приоритетов
- [ ] Лимит на конкретное приложение работает независимо от общего.
- [ ] Матрица: `alwaysAllowed` > пер-app > белый список > общий лимит.
- [ ] Расход за сегодня виден у родителя на «Лимиты приложений».

### 2.3 Бонусное время
- [ ] +15/+30 и произвольный бонус доезжают на детский.
- [ ] Бонус суммируется, досрочная отмена работает.
- [ ] Бонус на телефон и бонус на приложение — независимы.

### 2.4 Запрет приложений
- [ ] Запрещённое приложение блокируется всегда, независимо от лимитов.
- [ ] Оверлей показывает причину.
- [ ] Незапрещённые приложения ребёнок удаляет свободно (не ловим чужие удаления).

### 2.5 Блокировка интернета (VPN)
- [ ] Лимит исчерпан → интернет режется всем, кроме KidGuard и белого списка.
- [ ] Время есть → интернет у всех (pass-through).
- [ ] KidGuard-синхронизация выживает при lockdown.

### 2.6 Синхронизация
- [ ] Два родителя видят одну политику, LWW работает.
- [ ] Приглашение второго родителя по email.
- [ ] Селектор активного ребёнка.
- [ ] Приложения ребёнка (с иконками) видны у родителя — **иконки детские, не родительские**
      (свежий фикс: `StatisticsScreen` через `ChildAppsProvider`).

### 2.7 Попытки обхода — главное вехи 6А
- [ ] Спец. возможности → выключить KidGuard = **PIN-оверлей**.
- [ ] VPN-настройки → отключить = **PIN-оверлей**.
- [ ] Администраторы устройства → снять KidGuard = **PIN-оверлей** (и список, и карточка, и диалог).
- [ ] Удаление KidGuard = **PIN-оверлей**.
- [ ] Неверный PIN → уводит назад.
- [ ] Верный PIN → пропускает на 20 с (`UNLOCK_WINDOW_MS`), потом снова спрашивает.
- [ ] Перезагрузка → защита восстановилась.

### 2.8 Первичная настройка (свежий glassmorphism)
- [ ] Онбординг → выбор роли → привязка кодом → мастер разрешений → детский экран.
- [ ] Все экраны в новом дизайне, обе темы читаемы **на реальном экране** (не эмуляторе).
- [ ] Кнопка «Сгенерировать новый код» у привязанного ребёнка — на случай переустановки.
