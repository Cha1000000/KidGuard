Обзор всех вех — `roadmap.md`. Зависит от **вехи 2** (учёт времени, блокировка, белый список).

## Цель вехи

1. Лимиты на отдельные приложения (своё время на каждое приложение).
2. Бонусное время — дополнительное время на телефон целиком или на конкретное приложение.

Настройка пока **локальная** (удалённая выдача бонуса родителем — веха 4).

## Декомпозиция на шаги

### Шаг 3.1 — Модель пер-app лимитов
- Расширить политику: лимит на пакет (пакет → минут/день). Хранение в Room.
- **Критерий:** пер-app лимиты сохраняются и читаются.

### Шаг 3.2 — Поминутный учёт по приложениям
- Движок учёта (веха 2) считает время не только суммарно, но и **по каждому пакету** (детект
  активного приложения уже есть из вехи 1).
- **Критерий:** время по каждому приложению считается отдельно и корректно.

### Шаг 3.3 — Блокировка по пер-app лимиту
- Приложение блокируется оверлеем при исчерпании **своего** лимита, даже если общий дневной ещё
  не исчерпан. Логика приоритетов: белый список > бонус > пер-app лимит > общий лимит.
- **Критерий:** приложение с исчерпанным личным лимитом блокируется; другие доступны.

### Шаг 3.4 — Бонусное время
- Модель грантов: тип (весь телефон / конкретное приложение), объём (минуты), срок действия.
- Применение: активный грант снимает соответствующую блокировку до истечения.
- **Критерий:** выданный бонус временно снимает блок (телефона или приложения) и корректно истекает.

### Шаг 3.5 — Родительский UI (локально)
- Настройка пер-app лимитов (по списку приложений).
- Выдача бонуса (на телефон / на приложение, объём).
- **Критерий:** родитель локально задаёт пер-app лимиты и выдаёт бонус.

## Definition of Done

- Отдельные приложения блокируются по своим лимитам независимо от общего.
- Бонусное время временно снимает блок и корректно истекает.
- Приоритеты правил (белый список / бонус / пер-app / общий) работают предсказуемо.

## Открытые вопросы / на пересмотр перед стартом

- Точная матрица приоритетов правил — зафиксировать однозначно (чтобы не было конфликтов).
- Как визуализировать ребёнку остаток времени по приложениям (UX).
- Бонус разовый или может накапливаться / отменяться родителем досрочно.
