# AGENTS.md

Точка входа для AI-агентов в этом репозитории (Claude Code, Windsurf, Gemini и др.).
Основной контекст проекта — стек, архитектура, процесс, запреты — в `CLAUDE.md`.
Этот файл добавляет кросс-агентное правило **синхронизации со «вторым мозгом»**.

## Синхронизация со «Вторым мозгом» (начало и конец сессии)

У Владимира есть **карточка знаний** по этому проекту в его Obsidian-«втором мозге» (LLM Wiki по
методу Карпати), в отдельной папке **вне этого репозитория**:

- CachyOS: `~/YandexDisk/Obsidian/My Vault/projects/KidGuard.md`
- Подробная концепция/спека: `~/YandexDisk/Obsidian/My Vault/projects/KidGuard — концепция.md`
- На других ОС (macOS/Windows) путь к хранилищу другой — уточни у Владимира или смотри реестр
  `reference/Рабочие репозитории.md` в хранилище.

**В НАЧАЛЕ сессии:** сперва прочитай карточку [[KidGuard]] (и при необходимости концепцию),
чтобы поднять контекст прошлых сессий — решения, что уже сделано, открытые вопросы, рабочие
правила. Чтение — только чтение, по абсолютному пути. Если пути нет (другая ОС / хранилище не
синканулось) — спроси путь у Владимира, но работу не блокируй.

**В КОНЦЕ сессии** (после задачи, фикса или фичи): сам **предложи** занести итог в карточку
(молча не пиши). При согласии — допиши в раздел `## Решения и заметки` (новое сверху, с датой,
**на русском**): 1–3 строки — что изменили, ключевые решения, версия/ветка, что осталось;
проставь `[[связи]]`, обнови поле `updated:`. Правка — запись по абсолютному пути (запроси
разрешение). **Никогда** не коммить правку карточки в git этого репозитория — она живёт в
хранилище (синхронизирует Яндекс.Диск, git там запрещён).
