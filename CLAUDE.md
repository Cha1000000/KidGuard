# CLAUDE.md — KidGuard

Правила для AI-агента в этом репозитории. Читать в начале каждой сессии.

**KidGuard** — приложение родительского контроля для Android (Kotlin + Jetpack Compose),
самостоятельная альтернатива Google Family Link «под себя»: точный учёт реального экранного
времени и блокировка любых приложений (включая системные). Только для своей семьи, установка
`.apk` (не Google Play). Владелец — Владимир.

Полная концепция/спека — карточка `[[KidGuard — концепция]]` во «втором мозге» (см. `@AGENTS.md`).

## Стек

- **Язык:** Kotlin. **UI:** Jetpack Compose, Material 3. **Async:** Coroutines + Flow.
- **Архитектура:** Clean Architecture + MVVM (presentation / domain / data).
- **DI:** Hilt. **Локальные данные:** Room (источник истины на устройстве). **Сеть:** Retrofit + OkHttp.
- **Аутентификация:** Google Sign-In (Credential Manager).
- **Сборка:** Gradle (Kotlin DSL, version catalog `gradle/libs.versions.toml`).
- **Версии:** AGP 9.2.1, Kotlin 2.2.10, Compose BOM 2026.02. **minSdk 33**, targetSdk 36, compileSdk 37
  (compileSdk поднят с 36 до 37 — этого требует lifecycle 2.11.0; на совместимость устройств не влияет).
- **Package:** `ru.homelab.kidguard`.

> Библиотеки Hilt/Room/Retrofit/Google Sign-In в каркасе ещё НЕ подключены — добавляются по
> мере реализации. Сейчас в проекте только Compose-каркас (Empty Activity).

## Архитектура и ключевые решения

Три части системы: приложение **ребёнка**, приложение **родителей**, **бэкенд** на AdminVPS
для синхронизации. Роль («родитель»/«ребёнок») выбирается один раз при первичной настройке и
фиксируется навсегда (без повторного логина).

- **Контроль БЕЗ Device Owner** (его нельзя получить без ADB/сброса). Механизм на детском
  устройстве: **Accessibility Service** (детект активного приложения + оверлей-блокировка любых
  приложений, включая системные), **SYSTEM_ALERT_WINDOW** (оверлеи), **Device Admin** (защита от
  удаления), **VpnService** (блок интернета после лимита), **Foreground Service** (учёт времени).
- **Точный учёт = реальное экранное время:** списываем только когда экран включён
  (`PowerManager.isInteractive`) И разблокирован (`KeyguardManager`) И есть активное приложение.
  Фоновые процессы (музыка при погашенном экране) время НЕ расходуют.
- **Единая политика ребёнка:** правила — общий объект на сервере, оба равноправных родителя
  редактируют одну политику; на детский телефон приходит один согласованный набор правил.
- **Планируемые модули:** `core` (домен/правила), `parent`, `child` (исполнитель), `sync`.
  Пока проект одномодульный (`app`) — разобьём на модули осознанно на этапе реализации.

## Процесс (важно)

1. Перед кодом — обдумать запрос, предложить варианты, задать уточняющие вопросы.
   **НЕ угадывать**, чего хочет Владимир.
2. План нетривиальной фичи — отдельным `.md` в `docs/plans/` (на русском); реализованные планы
   переносить в `docs/plans/archive/`.
3. Код писать **только после явного одобрения** плана.
4. Идти небольшими проверяемыми шагами: фича → локальная сборка/прогон → отмашка → коммит.
5. **Всегда дожидаться ответа Владимира** на заданный вопрос — не действовать по своим
   гипотезам, если ответа пока нет.

## Запреты

- **НЕ коммить и НЕ пушь** без явной отмашки Владимира (сборку проверяет он локально). См. `@AGENTS.md`.

## Сборка и проверка

- Собрать debug: `./gradlew assembleDebug`
- Unit-тесты: `./gradlew testDebugUnitTest`
- Инструментальные (на устройстве/эмуляторе): `./gradlew connectedDebugAndroidTest`
- Lint: `./gradlew lint`
- JDK 17+, Android SDK (путь в `local.properties`).

## Стиль

- Общение, комментарии и планы — **на русском**; термины, код и названия API — на английском.
- **SOLID**, Clean Architecture. Отступы по конвенции Kotlin; табы и пробелы не смешивать.
- Compose: не раздувать composable-функции, дробить на мелкие, использовать `Preview`.
- Состояние поднимать в ViewModel, использовать `StateFlow`. Nullable-safety, без `!!`.
- **Не хардкодить UI-тексты** в `.kt`: все заголовки, лейблы, тексты кнопок и т.п. выносить в
  `res/values/strings.xml` и использовать через `stringResource(R.string.…)`.
